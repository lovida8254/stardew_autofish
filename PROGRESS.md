- **`main.py` frozen-aware**: `_runtime_dir()`로 exe 옆 config.json 우선 로드(+번들 fallback). exe에서도 튜닝값 적용.
- **exe 빌드**: `build_exe.bat`(PyInstaller `--onefile --windowed --add-data config.json`). 산출물 `dist\StardewAutoFishing.exe`(~89MB, opencv 포함). exe 옆에 config.json 두면 재빌드 없이 튜닝 가능. **exe/dist/build/*.spec는 .gitignore**(바이너리는 GitHub Release로 배포, 저장소에 커밋 안 함).

## 핵심 발견 (디버깅으로 알아낸 사실)

1. **입력**: 스타듀 바는 **게임 창이 포그라운드일 때만** 좌클릭 유지에 반응. → 입력 전 `focus_window()`로 자동 포커스(검증됨). 게임을 관리자 권한으로 켰다면 봇도 관리자 필요.
2. **물리(로그 `fishlog.csv`로 측정)**: 누름=바 **위로 빠름(-4.9px/frame)**, 뗌=**아래로 느림(+0.6px/frame)**. 비대칭(중력 약함). 방향은 정상(누름=위).
3. **물고기 검출 버그(가장 큰 원인)**: 기존 "어두운 것=물고기" 방식이 **트랙 바닥의 어두운 파란 물(H≈120)** 을 물고기로 오검출 → 봇이 계속 아래로만 감(press 9%). 
   - **수정**: 물고기 표식은 **청록색 H≈96**, 바닥 물은 파랑 H≈120 → **색상(Hue)으로 구분**. `detect()`를 청록 HSV 범위(`fish_hsv_lower/upper = [86,110,100]~[108,255,245]`) 기준으로 되돌림. 스냅샷 검증: 물고기 y를 정확히 추적(504→330 등).
4. **제어 방식 변천**: 뱅뱅(오버슈트) → PWM(안 올라감) → 예측(lookahead) → **윈도우 속도 예측**(현재). 검출 떨림에 강하도록 속도를 0.11초 구간으로 추정.

## 제어기 (`BarController` in main.py)
- 로직: `pred = 바중심 + 속도*lookahead`; `pred > (물고기+aim_offset)` 이면 누름(올림).
- 속도는 최근 `vel_window`초 구간으로 추정(노이즈 강건).
- `Mouse.set()`에 `min_pulse` 최소 펄스(너무 빠른 토글로 게임이 누름 놓치는 것 방지).

## 나중에 더 빠른 물고기로 어려워지면 (재튜닝 절차)
1. `start_assist.bat` 실행하고 미니게임 한 판 하면서 콘솔의 `bar=.. fish=.. vel=.. pred=..` 로그 확인.
2. 증상별 조정:
   - **바가 물고기보다 계속 아래(못 따라 올라감)** → `min_pulse` ↓, 또는 `lookahead` ↓.
   - **바가 위로 오버슈트/들썩** → `lookahead` ↑ 또는 `vel_window` ↑.
   - **급반전에서 뒤늦음** → `lookahead` ↑(예측 강화).
   - **바가 튀면**(다시 쪼개짐 의심) → `find_bar_center_y`의 조각 합산 임계나 `bar_min_area` 점검.
3. 값은 `config.json`에서 바로 바꾸고 재실행(코드 수정 불필요).

## 파일 구성
- **`start_assist.bat`**: ⭐ 최종 실행 런처(보조모드). 바탕화면 바로가기 `Stardew 자동낚시`가 이걸 가리킴.
- **`assist_run.py`**: ⭐ 보조모드 헤드리스 러너. 봇은 캐스팅/후킹 안 함(사용자가 함), 미니게임 바만 제어. ESC 종료.
- `main.py`: 설정/검출(`detect`, `find_bar_center_y`, `find_fish_blob`)/제어(`BarController`)/상태기계(`FishingBrain`)/입력(`Mouse`)/창포커스/CLI 봇.
- `gui.py`: Tkinter GUI + 백그라운드 엔진(`BotEngine`) + 캘리브레이션 오버레이 + 진단. (완전자동용 — 지금 운용엔 미사용)
- `calib_bar.py`: 바 물리 계측기(G,VMAX 실측용). 흔한 물고기 미니게임에서 사각파 입력→가속 G·속도한계 VMAX 추정→`PHYS_ANCHORS=[(G,VMAX)]` 한 줄 출력. (게임 필요)
- `sim_tune.py`: 오프라인 미니게임 시뮬레이터+파라미터 스윕(게임 불필요). SDV식 물리(대칭 G/VMAX/벽반발)+현실 물고기 5종+검출 노이즈+물리 앵커링. 상단 `PHYS_ANCHORS`가 (G,VMAX) 물리. **결론: 빠른 물고기는 파라미터 무관, 물리 한계 → calib_bar 실측 필요.**
- `verify_run.py`: 완전자동 헤드리스 러너(보류). `audio_bite.py`: 입질 소리 감지(현재 미사용, vision으로 대체).
